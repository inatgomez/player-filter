# Player Filter

A command-line tool that ingests multi-season StatsBomb open data, computes role-specific performance profiles for three positions — **fullback**, **defensive midfielder**, and **attacking midfielder** — and outputs a ranked shortlist of candidates with radar chart visualizations.

---

## What this tool does

Given a position and a minimum minutes threshold, the tool:

1. Reads from a locally cached player profiles dataset (no live API calls at runtime)
2. Applies the minutes filter and computes percentile ranks within the filtered population
3. Prints a ranked table of candidates to the terminal
4. Generates radar chart PNGs for the top 3 players, saved to `/output/`

The ranking defaults to mean percentile rank across all metrics for the selected position. An optional `--sort_by` argument overrides this with any single metric.

## Pipeline

The pipeline has three stages with a clean separation of responsibilities:

### Stage 1 — Ingest (`ingest.py`)

Fetches all raw event data from the StatsBomb open data API for the configured competitions and seasons (La Liga, 2017/18 through 2020/21). Writes a single flat parquet file to `data/raw_events.parquet`.

This script makes API calls. Run it once. If it fails mid-run, re-run — nothing is partially committed to disk.

```bash
python ingest.py
```

No arguments. Seasons and competition IDs are set inside the script.

---

### Stage 2 — Build profiles (`build_profiles.py`)

Reads `data/raw_events.parquet`. Computes all position-specific metrics, applies possession adjustment where applicable, normalizes to per-90 rates, and aggregates to one row per player per season. Writes `data/player_profiles.parquet`.

This script does all the metric computation. It does not call the API. If a metric definition changes, re-run this script without touching `ingest.py`.

```bash
python build_profiles.py
```

---

### Stage 3 — Filter and visualize (`filter.py` + `radar.py`)

Reads `data/player_profiles.parquet`. Applies role and minutes filters, computes percentile ranks within the filtered population, prints a ranked table to the terminal, and calls `radar.py` to generate radar PNGs for the top 3 players.

```bash
python filter.py --role fullback --min_minutes 900
python filter.py --role dm --min_minutes 900
python filter.py --role am --min_minutes 900
python filter.py --role fullback --min_minutes 900 --sort_by deep_progressions
python filter.py --role am --min_minutes 500 --sort_by xg
```

**Arguments:**

| Argument        | Required | Values                 | Description                               |
| --------------- | -------- | ---------------------- | ----------------------------------------- |
| `--role`        | Yes      | `fullback`, `dm`, `am` | Position group to filter                  |
| `--min_minutes` | Yes      | Integer                | Minimum minutes played across all seasons |
| `--sort_by`     | No       | Any metric column name | Override default mean-percentile ranking  |

When `--sort_by` is omitted, players are ranked by their mean percentile across all metrics for the selected role. This is the default composite score. The use of open data limits the possibility of using better scores such as On-Ball Value.

`radar.py` is not called directly. It is invoked internally by `filter.py` with the top 3 player IDs from the filtered output.

## Data source and scope

**Source:** [StatsBomb open data](https://github.com/statsbomb/open-data) — freely available, non-commercial use.

**Competitions:** La Liga only. Mixing competitions is avoided to keep defensive baselines consistent across players.

**Seasons:** 2017/18, 2018/19, 2019/20, 2020/21.

---

## Position taxonomy

Defined in full in `design_document.md`. Summary:

**Fullback** (`fullback`): Left Back, Right Back, Left Wing Back, Right Wing Back. Players are assigned their modal position within a season. Players whose modal position accounts for less than 60% of their positional events are excluded as tactically ambiguous.

**Defensive Midfielder** (`dm`): Defensive Midfield, Left Defensive Midfield, Right Defensive Midfield, Center Defensive Midfield. Central midfielders are excluded to keep pressing context and defensive expectations consistent.

**Attacking Midfielder** (`am`): Left Attacking Midfield, Right Attacking Midfield, Center Attacking Midfield. Wide midfielders and wingers are excluded to maintain a profile centered on central creativity rather than wide channel play.

## Metrics

Full definitions and justifications are in `docs/design_document.md`. The authoritative reference for metric definitions is the [Hudl StatsBomb event data glossary](https://support.hudl.com/s/article/event-data-glossary-player-metrics?language=en_US&topic=Statsbomb_Global_Football_Data_Glossary).

**Shared across positions:**

| Metric                | Normalization               | Positions  |
| --------------------- | --------------------------- | ---------- |
| Open play xG assisted | Per 90                      | FB, DM, AM |
| PAdj pressures        | Per 90, possession-adjusted | FB, DM, AM |
| Turnovers             | Per 90                      | FB, DM, AM |

**Fullback:**

| Metric                       | Normalization               |
| ---------------------------- | --------------------------- |
| Deep progressions            | Per 90                      |
| PAdj tackles & interceptions | Per 90, possession-adjusted |
| Tackle/Dribbled past %       | None (rate)                 |
| Aerial win %                 | None (rate)                 |
| Fouls committed              | Per 90                      |

**Defensive Midfielder:**

| Metric                       | Normalization               |
| ---------------------------- | --------------------------- |
| Deep progressions            | Per 90                      |
| PAdj tackles & interceptions | Per 90, possession-adjusted |
| Tackle/Dribbled past %       | None (rate)                 |
| Fouls won                    | Per 90                      |

**Attacking Midfielder:**

| Metric         | Normalization                        |
| -------------- | ------------------------------------ |
| xG             | Per 90                               |
| Shots          | Per 90                               |
| xG/Shot        | None (ratio; min. 20 shots required) |
| Touches in box | Per 90                               |
| Fouls won      | Per 90                               |

**Possession adjustment formula** (applied before per-90 normalization):

```
PAdj_x = x * 2 / (1 + exp(-10 * (possession - 0.5)))
```

Possession is calculated per match per team as: team passes / total passes in the match. This corrects for the fact that players on high-possession teams have fewer opportunities to record defensive actions.

## Known limitations

- Open data covers La Liga only. Percentile ranks reflect La Liga player populations; they are not cross-league benchmarks.
- StatsBomb open data withholds several features present in the licensed product (OBV, post-shot xG, Being Pressured Pass %). Metrics that require these are excluded.
- Position labels are StatsBomb-assigned. The inverted fullback role (a fullback who occupies central positions in possession) will still be tagged as Left/Right Back in this dataset and is not distinguished from a conventional fullback.
- xG/Shot requires a minimum of 20 shots. Players below this threshold are excluded from AM ranking by this metric.
- Cross-season aggregation treats all seasons equally. Player development trajectory is not modeled — a player's 2017/18 profile and 2020/21 profile contribute equally to the aggregate.

## Setup

```bash
git clone https://github.com/inatgomez/player-filter
cd player-filter
python -m venv .venv
source .venv/bin/activate       # Windows: .venv\Scripts\activate
pip install -r requirements.txt
python ingest.py
python build_profiles.py
python filter.py --role fullback --min_minutes 900
```

Radar PNGs for the top 3 players will be saved to `output/`.
